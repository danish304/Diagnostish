# Diagnostish

Портативная консольная утилита на C# для автоматизации первичной диагностики и сбора технических характеристик компьютера. Разработана специально для запуска с сервисных флеш-накопителей.

## 🚀 Основной функционал

*   **Сбор аппаратных данных:** Низкоуровневый опрос компонентов (процессор, оперативная память, видеокарты, накопители) через Windows Management Instrumentation (WMI).
*   **Многоуровневая отказоустойчивость (Fault Tolerance):** Интегрирован безопасный парсинг данных (`WmiParser`). Отсутствие или повреждение отдельных WMI-записей не приводит к падению утилиты — система продолжит сбор, зафиксировав предупреждения.
*   **Автоматизация прав доступа:** Встроенный манифест приложения (UAC) автоматически запрашивает права администратора для корректного чтения системных датчиков.
*   **Автономность (Self-Contained):** Утилита собирается в один исполняемый файл со всеми зависимостями. Работает на любом ПК с Windows без предварительной установки .NET.
*   **Информативность:** Результаты сканирования структурированы и выводятся в кастомный цветовой интерфейс консоли.

## 🏗 Архитектура и технологии

Проект спроектирован с соблюдением принципов SOLID (в частности, DIP и ISP), разделяя обязанности по слоям:

*   **Технологический стек:** Проект написан на языке C# (.NET 10).
*   **Models/ (`HWReport`, `OSReport`)** — независимые сущности (DTO) для хранения и транспортировки собранных данных, а также списков ошибок (`Errors`)
*   **Services/ (`CheckConfigurationWMI`)** — изолированная бизнес-логика сканирования компонентов на базе WMI. Реализует интерфейсы `IHWCheck` и `IOSCheck`, что позволяет легко заменить WMI на другую технологию (например, чтение Реестра) без изменения остального кода.
*   **Views/ (`PrintToConsole`)** — слой представления данных, построенный на интерфейсах `IPrintHW` и `IPrintOS` для гибкой подмены формата вывода (консоль, файл, GUI). 
*   **Controllers/ (`DiagnosticController`)** — управляющий слой, координирующий жизненный цикл приложения и потоки данных. Полностью отвязан от конкретных технологий сбора и вывода, работая исключительно через интерфейсы.
*   **Helpers/ (`WmiParser`)** — вспомогательный слой для безопасного приведения типов (`DBNull`, `null`, специфические CIM-даты) к безопасным типам данных C#.
*   **Program.cs** — чистая точка сборки (**Composition Root**), где создаются объекты и связываются зависимости. Не содержит логики.

## 📦 Как использовать

1. Скачайте последний релиз утилиты во вкладке **Releases**.
2. Поместите скомпилированный файл `.exe` на сервисную флешку.
3. Запустите утилиту на целевом ПК — она автоматически запросит права администратора, соберет базовые ТТХ и подготовит систему к дальнейшей диагностике.
