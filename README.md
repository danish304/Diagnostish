# Diagnostish

Портативная консольная утилита на C# для автоматизации первичной диагностики и сбора технических/программных характеристик компьютера. Разработана специально для запуска с сервисных флеш-накопителей.

## ⚖️ Лицензия
Этот проект распространяется под лицензией **MIT**. Вы можете свободно использовать, изменять и распространять код.

## 🚀 Основной функционал

*   **Сбор аппаратных данных:** Низкоуровневый опрос компонентов (процессор, оперативная память, видеокарты, накопители) через Windows Management Instrumentation (WMI).
*   **Многоуровневая отказоустойчивость:** Интегрирован безопасный парсинг данных (`Parser`). Отсутствие или повреждение отдельных WMI-записей не приводит к падению утилиты — система продолжит сбор, зафиксировав предупреждения и ошибки.
*   **Автоматизация прав доступа:** Встроенный манифест приложения (UAC) автоматически запрашивает права администратора для корректного чтения системных датчиков.
*   **Автономность:** Утилита собирается в один исполняемый файл со всеми зависимостями. Работает на любом ПК с Windows без предварительной установки .NET.
*   **Информативность:** Результаты сканирования структурированы и выводятся в кастомный цветовой интерфейс консоли.
*   **Логирование:** Все события и ошибки записываются в директорию `/logs` в формате текстового журнала. В случае возникновения проблем при диагностике, файл лога поможет быстро локализовать сбой.

## 🏗 Архитектура и технологии

Проект спроектирован с соблюдением принципов **SOLID**, обеспечивая высокую масштабируемость и легкость тестирования.

*   **Технологический стек:** Проект написан на языке C# (.NET 10).
*   **Models/ (`HWReport`, `OSReport`, `IssuesReport`)** — независимые сущности (DTO) для хранения и транспортировки собранных данных, а также списков ошибок.
*   **Services/ (`CheckPCConfigurationWMI`, `CheckOSConfigurationWMI`)** — изолированная бизнес-логика сканирования компонентов на базе WMI. Реализуют интерфейсы `IHWCheck` и `IOSCheck`, что позволяет легко заменить WMI на другую технологию (например, чтение Реестра) без изменения остального кода.
*   **Views/ (`PrintToConsole`)** — слой представления данных, построенный на интерфейсах `IPrintHW`, `IPrintOS` и `IUserInterface` для гибкой подмены формата вывода (консоль, файл, GUI). 
*   **Controllers/ (`DiagnosticController`)** — управляющий слой, координирующий жизненный цикл приложения и потоки данных. Полностью отвязан от конкретных технологий сбора и вывода, работая исключительно через интерфейсы.
*   **Helpers/ (`Parser`, `SafeExecutor`, `WMISettings`)** — вспомогательный слой для безопасного приведения типов (`DBNull`, `null`, специфические CIM-даты) к безопасным типам данных C#, централизованного управления безопасным выполнением WMI-запросов с атомарной очисткой коллекций, а также хранения глобальных конфигурационных параметров утилиты, таких как таймауты запросов.
*   **Program.cs** — точка сборки (**Composition Root**), где создаются объекты и связываются зависимости.
  
## 📋 Требования

* ОС: Windows 7 SP1 и выше.
* Права: Администратор (требуется для доступа к WMI-сенсорам).
* .NET Runtime: Встроен (Self-Contained билд).
  
## 📦 Как использовать

*   Скачайте последний релиз утилиты во вкладке **Releases**.
*   Поместите скомпилированный файл `.exe` на сервисную флешку.
*   Запустите утилиту на целевом ПК — она автоматически запросит права администратора, получит диагностические данные и выведет их в соответсвующем интерфейсе.
